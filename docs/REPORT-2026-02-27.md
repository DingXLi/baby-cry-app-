#### 后端 (Node.js + Express)
- ✅ `backend/package.json` - 依赖配置
- ✅ `backend/.env.example` - 环境变量模板
- ✅ `backend/src/index.js` - 服务器入口
- ✅ `backend/src/models/User.js` - 用户模型
- ✅ `backend/src/models/CryRecord.js` - 哭声记录模型

**已配置依赖:**
- Express (Web 框架)
- Sequelize (ORM)
- PostgreSQL (数据库)
- JWT (认证)
- Helmet (安全)
- Winston (日志)

---

#### 机器学习 (Python + TensorFlow)
- ✅ `ml/requirements.txt` - Python 依赖
- ✅ `ml/models/cry_classifier.py` - 模型定义
- ✅ `ml/training/train.py` - 训练脚本

**模型架构:**
- 输入：128x128 Mel 频谱图
- 3 个 CNN 块 + BatchNorm + Dropout
- 输出：4 类分类 (饥饿/困倦/不适/正常)
- 支持导出为 TensorFlow Lite (移动端部署)

**依赖库:**
- TensorFlow 2.15+
- Librosa (音频处理)
- ONNX (模型转换)

## 📊 技术决策

### 1. 数据集策略
- **Phase 1:** 使用 ESC-50 数据集做哭声检测原型
- **Phase 2:** 寻找 Kaggle 哭声分类数据集
- **Phase 3:** 自行收集数据优化模型

**待办:** Li 需要去 Kaggle 搜索合适的数据集

---

### 2. 后端架构
- **数据库:** PostgreSQL (关系型，适合结构化数据)
- **ORM:** Sequelize (TypeScript 友好，迁移方便)
- **认证:** JWT (无状态，适合移动端)
- **部署:** Docker + Railway/Render (后续)

---

### 3. 移动端架构
- **框架:** React Native + Expo (快速开发，跨平台)
- **ML 推理:** TensorFlow Lite (本地运行，保护隐私)
- **状态管理:** 先用 React Context，后期考虑 Redux
- **导航:** React Navigation v6
