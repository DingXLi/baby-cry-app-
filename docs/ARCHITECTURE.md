# 🦞 婴儿哭声识别 - 系统架构

## 项目概述

**目标：** 开发一个能识别婴儿哭声类型并通知父母的移动应用

**核心功能：**
- 实时音频监控
- 哭声检测和分类（饥饿/困倦/不适/正常）
- 即时通知推送
- 历史记录和统计

## 核心模块

### 1. 音频采集模块 (mobile)
**职责：** 后台录音、音频预处理

**关键技术点：**
- 后台持续录音权限
- 音频格式：16kHz, 单声道，PCM
- 静音检测（减少无效处理）
- 隐私保护（本地处理优先）

**待确认问题：**
- [ ] 需要支持 iOS 和 Android 双平台吗？
- [ ] 是否需要后台持续监听（耗电考虑）？

---

### 2. ML 推理模块 (mobile + ml)
**职责：** 哭声检测 + 分类

**模型选择：**
- 输入：音频频谱图 (Mel Spectrogram)
- 模型：轻量级 CNN 或 MobileNetV3
- 输出：4 类分类 (饥饿/困倦/不适/正常)
- 推理时间：< 100ms

**待确认问题：**
- [ ] 有现成的哭声数据集吗？还是需要自己收集？
- [ ] 需要支持多婴儿识别吗？

---

### 3. 通知模块 (mobile)
**职责：** 哭声检测后推送通知

**功能：**
- 本地推送（即时）
- 通知类型区分（紧急程度）
- 可配置的通知规则

---

### 4. 数据存储 (mobile)
**职责：** 本地存储历史记录

**方案：**
- SQLite / WatermelonDB
- 存储：时间、哭声类型、音频片段（可选）
- 隐私：数据本地存储，不上传云端（除非用户选择）
